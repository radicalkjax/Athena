1//! A zero allocation library for parsing & writing a bunch of packet based protocols (EthernetII, IPv4, IPv6, UDP, TCP ...).
2//!
3//! Currently supported are:
4//! * Ethernet II
5//! * IEEE 802.1Q VLAN Tagging Header
6//! * IPv4
7//! * IPv6 (supporting the most common extension headers, but not all)
8//! * UDP
9//! * TCP
10//! * ICMP & ICMPv6 (not all message types are supported)
11//!
12//! # Usage
13//!
14//! Add the following to your `Cargo.toml`:
15//!
16//! ```toml
17//! [dependencies]
18//! etherparse = "0.14"
19//! ```
20//!
21//! # What is etherparse?
22//!
23//! Etherparse is intended to provide the basic network parsing functions that allow for easy analysis, transformation or generation of recorded network data.
24//!
25//! Some key points are:
26//!
27//! * It is completely written in Rust and thoroughly tested.
28//! * Special attention has been paid to not use allocations or syscalls.
29//! * The package is still in development and can & will still change.
30//! * The current focus of development is on the most popular protocols in the internet & transport layer.
31//!
32//! # How to parse network packages?
33//! Etherparse gives you two options for parsing network packages automatically:
34//!
35//! ## Slicing the packet
36//! Here the different components in a packet are separated without parsing all their fields. For each header a slice is generated that allows access to the fields of a header.
37//! ```
38//! # use etherparse::{SlicedPacket, PacketBuilder};
39//! # let builder = PacketBuilder::
40//! #    ethernet2([1,2,3,4,5,6],     //source mac
41//! #               [7,8,9,10,11,12]) //destination mac
42//! #    .ipv4([192,168,1,1], //source ip
43//! #          [192,168,1,2], //destination ip
44//! #          20)            //time to life
45//! #    .udp(21,    //source port
46//! #         1234); //destination port
47//! #    //payload of the udp packet
48//! #    let payload = [1,2,3,4,5,6,7,8];
49//! #    //get some memory to store the serialized data
50//! #    let mut packet = Vec::<u8>::with_capacity(
51//! #                            builder.size(payload.len()));
52//! #    builder.write(&mut packet, &payload).unwrap();
53//! match SlicedPacket::from_ethernet(&packet) {
54//!     Err(value) => println!("Err {:?}", value),
55//!     Ok(value) => {
56//!         println!("link: {:?}", value.link);
57//!         println!("vlan: {:?}", value.vlan);
58//!         println!("net: {:?}", value.net); // contains ip
59//!         println!("transport: {:?}", value.transport);
60//!     }
61//! }
62//! ```
63//! This is the faster option if your code is not interested in all fields of all the headers. It is a good choice if you just want filter or find packages based on a subset of the headers and/or their fields.
64//!
65//! Depending from which point downward you want to slice a package check out the functions:
66//!
67//! * [`SlicedPacket::from_ethernet`] for parsing from an Ethernet II header downwards
68//! * [`SlicedPacket::from_ether_type`] for parsing a slice starting after an Ethernet II header
69//! * [`SlicedPacket::from_ip`] for parsing from an IPv4 or IPv6 downwards
70//!
71//! In case you want to parse cut off packets (e.g. packets returned in in ICMP message) you can use the "lax" parsing methods:
72//!
73//! * [`LaxSlicedPacket::from_ethernet`] for parsing from an Ethernet II header downwards
74//! * [`LaxSlicedPacket::from_ether_type`] for parsing a slice starting after an Ethernet II header
75//! * [`LaxSlicedPacket::from_ip`] for parsing from an IPv4 or IPv6 downwards
76//!
77//! ## Deserializing all headers into structs
78//!
79//! This option deserializes all known headers and transfers their contents to header structs.
80//! ```rust
81//! # use etherparse::{PacketHeaders, PacketBuilder};
82//! # let builder = PacketBuilder::
83//! #    ethernet2([1,2,3,4,5,6],     //source mac
84//! #               [7,8,9,10,11,12]) //destination mac
85//! #    .ipv4([192,168,1,1], //source ip
86//! #          [192,168,1,2], //destination ip
87//! #          20)            //time to life
88//! #    .udp(21,    //source port
89//! #         1234); //destination port
90//! #    //payload of the udp packet
91//! #    let payload = [1,2,3,4,5,6,7,8];
92//! #    //get some memory to store the serialized data
93//! #    let mut packet = Vec::<u8>::with_capacity(
94//! #                            builder.size(payload.len()));
95//! #    builder.write(&mut packet, &payload).unwrap();
96//! match PacketHeaders::from_ethernet_slice(&packet) {
97//!     Err(value) => println!("Err {:?}", value),
98//!     Ok(value) => {
99//!         println!("link: {:?}", value.link);
100//!         println!("vlan: {:?}", value.vlan);
101//!         println!("net: {:?}", value.net); // contains ip
102//!         println!("transport: {:?}", value.transport);
103//!     }
104//! }
105//! ```
106//! This option is slower then slicing when only few fields are accessed. But it can be the faster option or useful if you are interested in most fields anyways or if you want to re-serialize the headers with modified values.
107//!
108//! Depending from which point downward you want to unpack a package check out the functions
109//!
110//! * [`PacketHeaders::from_ethernet_slice`] for parsing from an Ethernet II header downwards
111//! * [`PacketHeaders::from_ether_type`] for parsing a slice starting after an Ethernet II header
112//! * [`PacketHeaders::from_ip_slice`] for parsing from an IPv4 or IPv6 downwards
113//!
114//! In case you want to parse cut off packets (e.g. packets returned in in ICMP message) you can use the "lax" parsing methods:
115//!
116//! * [`LaxPacketHeaders::from_ethernet`] for parsing from an Ethernet II header downwards
117//! * [`LaxPacketHeaders::from_ether_type`] for parsing a slice starting after an Ethernet II header
118//! * [`LaxPacketHeaders::from_ip`] for parsing from an IPv4 or IPv6 downwards
119//!
120//! ## Manually slicing only one packet layer
121//!
122//! It is also possible to only slice one packet layer:
123//!
124//! * [`Ethernet2Slice::from_slice_without_fcs`] & [`Ethernet2Slice::from_slice_with_crc32_fcs`]
125//! * [`SingleVlanSlice::from_slice`] & [`DoubleVlanSlice::from_slice`]
126//! * [`IpSlice::from_slice`] & [`LaxIpSlice::from_slice`]
127//! * [`Ipv4Slice::from_slice`] & [`LaxIpv4Slice::from_slice`]
128//! * [`Ipv6Slice::from_slice`] & [`LaxIpv6Slice::from_slice`]
129//! * [`UdpSlice::from_slice`] & [`UdpSlice::from_slice_lax`]
130//! * [`TcpSlice::from_slice`]
131//! * [`Icmpv4Slice::from_slice`]
132//! * [`Icmpv6Slice::from_slice`]
133//!
134//! The resulting data types allow access to both the header(s) and the payload of the layer
135//! and will automatically limit the length of payload if the layer has a length field limiting the
136//! payload (e.g. the payload of IPv6 packets will be limited by the "payload length" field in
137//! an IPv6 header).
138//!
139//! ## Manually slicing & parsing only headers
140//!
141//! It is also possible just to parse headers. Have a look at the documentation for the
142//! following \[NAME\]HeaderSlice.from_slice methods, if you want to just slice the header:
143//!
144//! * [`Ethernet2HeaderSlice::from_slice`]
145//! * [`SingleVlanHeaderSlice::from_slice`]
146//! * [`DoubleVlanHeaderSlice::from_slice`]
147//! * [`Ipv4HeaderSlice::from_slice`]
148//! * [`Ipv4ExtensionsSlice::from_slice`]
149//! * [`Ipv6HeaderSlice::from_slice`]
150//! * [`Ipv6ExtensionsSlice::from_slice`]
151//! * [`Ipv6RawExtHeaderSlice::from_slice`]
152//! * [`IpAuthHeaderSlice::from_slice`]
153//! * [`Ipv6FragmentHeaderSlice::from_slice`]
154//! * [`UdpHeaderSlice::from_slice`]
155//! * [`TcpHeaderSlice::from_slice`]
156//!
157//! And for deserialization into the corresponding header structs have a look at:
158//!
159//! * [`Ethernet2Header::read`] & [`Ethernet2Header::from_slice`]
160//! * [`SingleVlanHeader::read`] & [`SingleVlanHeader::from_slice`]
161//! * [`DoubleVlanHeader::read`] & [`DoubleVlanHeader::from_slice`]
162//! * [`IpHeaders::read`] & [`IpHeaders::from_slice`]
163//! * [`Ipv4Header::read`] & [`Ipv4Header::from_slice`]
164//! * [`Ipv4Extensions::read`] & [`Ipv4Extensions::from_slice`]
165//! * [`Ipv6Header::read`] & [`Ipv6Header::from_slice`]
166//! * [`Ipv6Extensions::read`] & [`Ipv6Extensions::from_slice`]
167//! * [`Ipv6RawExtHeader::read`] & [`Ipv6RawExtHeader::from_slice`]
168//! * [`IpAuthHeader::read`] & [`IpAuthHeader::from_slice`]
169//! * [`Ipv6FragmentHeader::read`] & [`Ipv6FragmentHeader::from_slice`]

175//! # How to generate fake packet data?
176//!
177//! ## Packet Builder
178//!
179//! The PacketBuilder struct provides a high level interface for quickly creating network packets. The PacketBuilder will automatically set fields which can be deduced from the content and compositions of the packet itself (e.g. checksums, lengths, ethertype, ip protocol number).
180//!
181//! [Example:](https://github.com/JulianSchmid/etherparse/blob/0.14.3/examples/write_udp.rs)
182//! ```rust
183//! use etherparse::PacketBuilder;
184//!
185//! let builder = PacketBuilder::
186//!     ethernet2([1,2,3,4,5,6],     //source mac
187//!                [7,8,9,10,11,12]) //destination mac
188//!     .ipv4([192,168,1,1], //source ip
189//!           [192,168,1,2], //destination ip
190//!           20)            //time to life
191//!     .udp(21,    //source port
192//!          1234); //destination port
193//!
194//! //payload of the udp packet
195//! let payload = [1,2,3,4,5,6,7,8];
196//!
197//! //get some memory to store the result
198//! let mut result = Vec::<u8>::with_capacity(builder.size(payload.len()));
199//!
200//! //serialize
201//! //this will automatically set all length fields, checksums and identifiers (ethertype & protocol)
202//! //before writing the packet out to "result"
203//! builder.write(&mut result, &payload).unwrap();
204//! ```
205//!
206//! There is also an [example for TCP packets](https://github.com/JulianSchmid/etherparse/blob/0.14.3/examples/write_tcp.rs) available.
207//!
208//! Check out the [PacketBuilder documentation](struct.PacketBuilder.html) for more information.
209//!
210//! ## Manually serializing each header
211//!
212//! Alternatively it is possible to manually build a packet
213//! ([example](https://github.com/JulianSchmid/etherparse/blob/0.14.3/examples/write_ipv4_udp.rs)).
214//! Generally each struct representing a header has a "write" method that allows it to be
215//! serialized. These write methods sometimes automatically calculate checksums and fill them
216//! in. In case this is unwanted behavior (e.g. if you want to generate a packet with an invalid
217//! checksum), it is also possible to call a "write_raw" method that will simply serialize the data
218//! without doing checksum calculations.
219//!
220//! Read the documentations of the different methods for a more details:
221//!
222//! * [`Ethernet2Header::to_bytes`] & [`Ethernet2Header::write`]
223//! * [`SingleVlanHeader::to_bytes`] & [`SingleVlanHeader::write`]
224//! * [`DoubleVlanHeader::to_bytes`] & [`DoubleVlanHeader::write`]
225//! * [`Ipv4Header::to_bytes`] & [`Ipv4Header::write`] & [`Ipv4Header::write_raw`]
226//! * [`Ipv4Extensions::write`]
227//! * [`Ipv6Header::to_bytes`] & [`Ipv6Header::write`]
228//! * [`Ipv6Extensions::write`]
229//! * [`Ipv6RawExtHeader::to_bytes`] & [`Ipv6RawExtHeader::write`]
230//! * [`IpAuthHeader::to_bytes`] & [`IpAuthHeader::write`]
231//! * [`Ipv6FragmentHeader::to_bytes`] & [`Ipv6FragmentHeader::write`]
232//! * [`UdpHeader::to_bytes`] & [`UdpHeader::write`]
233//! * [`TcpHeader::to_bytes`] & [`TcpHeader::write`]
234//! * [`Icmpv4Header::to_bytes`] & [`Icmpv4Header::write`]
235//! * [`Icmpv6Header::to_bytes`] & [`Icmpv6Header::write`]
